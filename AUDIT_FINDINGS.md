## 🔴 КРИТИЧЕСКИЕ ОШИБКИ (Нужно исправить немедленно)

### 1. **cart.js:309 — Доставка не добавляется к сумме заказа**
- **Файл:** `js/cart.js`, строка 309
- **Проблема:** Переменная `delivery` рассчитывается, но никогда не добавляется к `total`
- **Код:**
  ```javascript
  const delivery = (!isEmpty && baseSum < DELIVERY_THRESHOLD) ? DELIVERY_COST : 0
  const total = sum  // ❌ BUG: delivery не включена
  ```
- **Следствие:** Клиенты не будут заряжены за доставку, если сумма меньше 5000 руб
- **Исправление:** `const total = sum + delivery`

### 2. **OrderController.php:180 — Запутанная логика расчета baseTotal**
- **Файл:** `app/Controller/OrderController.php`, строка 180
- **Проблема:** Оператор `??` с `+` создает запутанную логику из-за precedence
- **Код:**
  ```php
  $baseTotal = (float)($_SESSION['base_total'] ?? $totalSum + $discountAmount);
  ```
- **Проблема:** Хотя технически работает, это очень сложно читать и легко ошибиться
- **Исправление:** Добавить скобки или хранить базовую сумму отдельно в сессии

## 🟠 ВАЖНЫЕ ПРОБЛЕМЫ (Нужно исправить перед merge)

### 3. **cart.js:486 — Запутанная логика пересчета скидки купона**
- **Файл:** `js/cart.js`, строка 486
- **Проблема:** Код добавляет `coupon_discount` обратно к `totalPrice()`, который уже его вычел
- **Код:**
  ```javascript
  const sum = this.store.totalPrice() + this.store.coupon_discount
  ```
- **Следствие:** Логика работает, но очень запутана для понимания
- **Исправление:** Добавить комментарий или создать метод `baseTotalPrice()`

### 4. **admin/login.php:20 — MD5 на IP адрес для rate limit**
- **Файл:** `admin/login.php`, строка 20
- **Проблема:** Использование MD5 для ключа rate limit ненужно
- **Код:**
  ```php
  check_rate_limit('admin_login_' . md5($_SERVER['REMOTE_ADDR'] ?? ''), 10, 300)
  ```
- **Следствие:** Усложняет отладку, не дает никакой безопасности
- **Исправление:** Использовать IP напрямую: `'admin_login_' . ($_SERVER['REMOTE_ADDR'] ?? 'unknown')`

### 5. **config/app.php:16 — Setup токен остается в коде**
- **Файл:** `config/app.php`, строка 16
- **Проблема:** `setup_token` имеет значение по умолчанию, которое видно в коде
- **Код:**
  ```php
  'setup_token' => 'r12sfdh34wrt24g3wffsd34653ghd',  // ← установите сюда уникальный токен
  ```
- **Следствие:** Если кто-то найдет этот файл, может запустить setup
- **Исправление:** Изменить на случайный токен после первого запуска или удалить setup.php
