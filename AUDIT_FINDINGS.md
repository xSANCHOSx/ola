## 🔴 КРИТИЧНІ БАГИ (Потрібно виправити негайно)

### 1. **cart.js:309 — Доставка не додається до суми замовлення**
- **Файл:** `js/cart.js`, рядок 309
- **Проблема:** Змінна `delivery` розраховується, але ніколи не додається до `total`
- **Код:**
  ```javascript
  const delivery = (!isEmpty && baseSum < DELIVERY_THRESHOLD) ? DELIVERY_COST : 0
  const total = sum  // ❌ BUG: delivery не включена
  ```
- **Наслідок:** Клієнти не будуть заряджені за доставку, якщо сума менше 5000 руб
- **Виправлення:** `const total = sum + delivery`

### 2. **OrderController.php:180 — Заплутана логіка розрахунку baseTotal**
- **Файл:** `app/Controller/OrderController.php`, рядок 180
- **Проблема:** Оператор `??` з `+` створює заплутану логіку через precedence
- **Код:**
  ```php
  $baseTotal = (float)($_SESSION['base_total'] ?? $totalSum + $discountAmount);
  ```
- **Проблема:** Хоча технічно працює, це дуже важко читати і легко помилитися
- **Виправлення:** Додати дужки або зберігати базову суму окремо в сесії

## 🟠 ВАЖЛИВІ ПРОБЛЕМИ (Потрібно виправити перед merge)

### 3. **cart.js:486 — Заплутана логіка перерахунку знижки купона**
- **Файл:** `js/cart.js`, рядок 486
- **Проблема:** Код додає `coupon_discount` назад до `totalPrice()`, який вже його відняв
- **Код:**
  ```javascript
  const sum = this.store.totalPrice() + this.store.coupon_discount
  ```
- **Наслідок:** Логіка працює, але дуже заплутана для розуміння
- **Виправлення:** Додати коментар або створити метод `baseTotalPrice()`

### 4. **admin/login.php:20 — MD5 на IP адресу для rate limit**
- **Файл:** `admin/login.php`, рядок 20
- **Проблема:** Використання MD5 для rate limit ключа непотрібне
- **Код:**
  ```php
  check_rate_limit('admin_login_' . md5($_SERVER['REMOTE_ADDR'] ?? ''), 10, 300)
  ```
- **Наслідок:** Ускладнює дебаг, не дає жодної безпеки
- **Виправлення:** Використовувати IP напряму: `'admin_login_' . ($_SERVER['REMOTE_ADDR'] ?? 'unknown')`

### 5. **config/app.php:16 — Setup токен залишається в коді**
- **Файл:** `config/app.php`, рядок 16
- **Проблема:** `setup_token` має дефолтне значення, яке видно в коді
- **Код:**
  ```php
  'setup_token' => 'r12sfdh34wrt24g3wffsd34653ghd',  // ← встановіть сюди унікальний токен
  ```
- **Наслідок:** Якщо хто-то знайде цей файл, може запустити setup
- **Виправлення:** Змінити на випадковий токен після першого запуску або видалити setup.php
